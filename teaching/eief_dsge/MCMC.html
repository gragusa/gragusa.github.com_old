Monte Carlo Markov Chain in Julia

Julia provides some very well written packages for Baysian estimation.

The most useful are Lora and Mamba.

Lora provides a generic engine for Markov Chain Monte Carlo (MCMC) inference. It is undergoing a major upgrade and some feature stands up, such as the an=bility to use automotic differentiation for samplers that require the gradient of the log-target.

Mamba is an open platform for the implementation and application of MCMC methods to perform Bayesian analysis. It provides a framework for (1) specification of hierarchical models through stated relationships between data, parameters, and statistical distributions; (2) block-updating of parameters with samplers provided, defined by the user, or available from other packages; (3) execution of sampling schemes; and (4) posterior inference. It is intended to give users access to all levels of the design and implementation of MCMC simulators to particularly aid in the development of new methods.

A standard problem

We will illustrate how to use Lora and Mamba for obtaining draws from a posterior distribution by using a simple example. Once we have understood the logic we will move to a Kalman Filter based likelihood.

The log-target

The log-target is the logarithm of the numerator of the posterior distribution. Using the standard notation, if \[post(\theta) = \gamma(\theta)p(\theta)/Z\] is the posterior, the log-target is given by \[logtarget(\theta) = \log \gamma(\theta) + \log p(\theta)\].

Let's consider the linear model with \(t\) distributed errors. The data \(\{y_i, x_i\}\), \(i=1,\ldots,n\) are realization of an iid sequence of random vector \((Y_i, X_i)\) whose distribution is \[Y_i | X_i = x_i \sim t(\nu, \alpha + \beta x_i, \sigma^2),\] where \(t(\nu, \mu, \sigma^2)\) denote the distribution of a random variable so defined \[Z = \mu + \frac{\sigma\epsilon}/{sqrt{\frac{ \nu}{\nu-2}}}, \quad \epsilon \sim t(\nu),\quad \nu > 2,\] where \(t(\nu)\) denotes the t-student distribution with \(\nu\) degrees of freedom.

The likelihood function for this model is (let \(\theta = (\alpha, \beta, \sigma)\)) \[ \gamma(\theta) = \sigma^{-n} \prod_{i=1}^n \left( 1+\frac{(y_i-\alpha-\beta x_i)^2}{\nu\sigma^2}\right)^{-(\nu+1)/2}. \] We use an improper prior \[ p(\theta) \propto \sigma^{-1} \] giving thus the following log-target \[ logtarget(\theta) = -(n+1)\log \sigma - (\nu+1)/2 \sum_{i=1}^n \log\left( 1+\frac{(y_i-\alpha-\beta x_i)^2}{\nu\sigma^2}\right). \]

We start by generating data according to the model. We use the following parameterization

using Distributions
ν = 3; n = 100; α = 0.0; β = 0.5; σ² = 1.0
d = TDist(ν)
srand(123333)
x = rand(d, n)./sqrt(ν/(ν-2))
y = rand(d, n)./sqrt(ν/(ν-2)) .* sqrt(σ²) + α + β*x

The OLS estimates of \((\alpha, \beta)\) are

bols = [ones(n) x]\y

the estimated variance

s² = sumabs2(y-[ones(n) x]*bols)/(n-1)

We will use these values to initialize the MCMC algorithm.

We now need to code the function that evaluates the log-target at \(\theta\).

function logtarget(theta)
   n  = length(y)
   α  = theta[1]
   β  = theta[2]
   σ² = theta[3]
   -(n+1)*log(sqrt(σ²)) - (ν+1)/2*sum(log(1+(y-α-β*x).^2/(ν*σ²)))
end

This function would not however work. The problem is that both \(\sigma\) is restricted to be positive. In order to avoid problems with eventual proposed negative values, we will reparameterize the log-target in terms of \(\theta = (\alpha, \beta, \log(\sigma))\). This amount rewrite the log-target as follows.

function logtarget(theta)
   n  = length(y)
   α  = theta[1]
   β  = theta[2]
   σ² = exp(theta[3])
   -(n+1)*log(sqrt(σ²)) - (ν+1)/2*sum(log(1+(y-α-β*x).^2/(ν*σ²)))
end

using Mamba
## Data
data = Dict(
  :x => x,
  :y => y
)

function plogtarget(theta)
   n  = length(y)
   α  = theta[1]
   β  = theta[2]
   σ  = exp(theta[3])
   -(n+1)*log(σ) - (ν+1)/2*sum(log(1+(data[:y]-α-β*data[:x]).^2/(ν*σ^2)))
end


n = 50000
burnin = 1000

sim = Chains(n, 3, names = ["α", "β", "σ²"]);

theta = RWMVariate([bols; log(sqrt(s²))], [0.1, 0.1, 0.1], plogtarget,
                   proposal = Normal)

for i in 1:n
  sample!(theta)
  sim[i, :, 1] = [theta[1:2]; exp(theta[3])]
end

Kalman Filter

Consider a simplified version of the SS:

\begin{align*} x_{t+1} & =g_{0}+g_{1}x_{t}+m_{1}\varepsilon_{t+1},\quad\varepsilon_{t}\sim N(0,\sigma_{\varepsilon}^{2})\\ y_{t} & =h_{0}+h_{1}x_{t}+\eta_{t},\quad\eta_{t}\sim N(0,\sigma_{\eta}^{2}) \end{align*}

The objective is to estimate \[ \theta=\{g_{0},g_{1},m_{1},\sigma_{\varepsilon},h_{0},h_{1},\sigma_{\eta}\} \] from a realization of the observable \(\{y\}_{t=1}^{T}\). This can be done in \(3\) steps:

  1. Apply the Kalman's filter
  2. Calculate the likelihood functions
  3. Apply a the MH algorithm to get the posterior distribution of the parameter

Likelihood

The function kfloglik evaluate the log-likelihood of the space state model. While this function will only work for scalar SS model, for our purposes will suffice.

function kfloglik(x0, s0, g0, g1, m1, sigmae, h0, h1,  sigmaeta, y)
    T = size(y, 1)
    ## x_{t|t}
    filt  = zeros((T+1),1)
    Ptt   = zeros((T+1),1)
    Ptt[1,1] = s0
    pred = zeros((T+1),1)
    Pt = zeros((T+1),1)

    for j=1:T
        ## Predictions step
        pred[j,1] = g0+g1*filt[j,1]
        Pt[j,1] = g1*Ptt[j,1]*g1+m1^2*sigmae^2
        ## Updating step
        K = Pt[j,1]*h1*(h1^2*Pt[j,1]+sigmaeta^2)^(-1)
        filt[j+1,1] = pred[j,1] + K*(y[j,1]-h0-h1*pred[j,1])
        Ptt[j+1,1] = Pt[j,1] - K*(h1*Pt[j,1])
    end

    mu = h0+h1*pred
    sd = h1*Pt*h1+sigmaeta.^2
    -T*log(2*pi)/2+sum(-log(sd[1:T])/2-(y-mu[1:T]).^2./(2*sd[1:T]))
end

Restricted Log-likelihood

To make things a little more interesting we are going to assume that we need to estimate only a subset of \(\theta\). In particular, we will estimate only \(g_0, g_1, \sigma_{\epsilon}\). In order to do so, it will suffice to define logtarget to take as input a \(3\times 1\) vector and call kfloglik with the fixed values for the other parameters. In particular, we will set \(h_0 = 0\), \(h_1 = 1\) and \(\sigma_{\eta}=0.3\).

function logtarget(theta)
    ## theta = (g0, g1, sigmae)
    g0 = theta[1]
    g1 = exp(theta[2])/(1+exp(theta[2]));
    sigmae = exp(theta[3])
    s0 = sigmae^2/(1-g1^2)
    kfloglik(0.0, s0, g0, g1, 1.0, sigmae, 0, 1, 0.3, y)
end

Notice that we are assuming flat prior on \(\theta\). Changing the cose to allow for informative priors is a matter of summing \(\log p(\theta)\) to the value returned by the call to kfloglik.

Maximum likelihood

Maximizing the log-likelihood can be done by optimize from the package Optim.

using Optim

optimize takes many arguments and allows optimizing a function with different algorithms (BFGS, L-BFGS, Nelder Mean, etc.).

A typical call to optimize is

optimize(f, par0, method)

where f is the function to be optimized and par0 is the vector of starting value for the selected method

Importantly, optimize will assume the f is to be minimized. So, if we were to do this

optimize(logtarget, [0., .5, .5], BFGS())

will be calculating the minimum likelihood estimator!

A way to circumvent this problem is to define the function to be minimized as the negative log-target:

negative_logtarget(theta) = -logtarget(theta)

and then optimize over negative_logtarget

out = optimize(negative_logtarget, [0, .5, .5])

Optim has many functions to extract information from the object out.

julia> Optim.minimum(out)

118.22068348987753
julia> init0 = Optim.minimizer(out)

  0.00896742

init0 contains the ML estimates which we will use to start the MH algorithm. chain. values can be used to initialized the MCMC chain. But we also need a value for the hessian to properly scale the proposal distribution.

using Calculus
invH = inv(hessian(negative_logtarget, Optim.minimizer(out)))

using Mamba
## Data
data = Dict(
  :y => y
)


n = 50000
burnin = 1000

sim = Chains(n, 3, names = ["g0", "g1", "sigmae"]);


function logtarget(theta)
    ## theta = (g0, g1, sigmae)
    g0 = theta[1]
    g1 = exp(theta[2])/(1+exp(theta[2]));
    sigmae = exp(theta[3])
    s0 = sigmae^2/(1-g1^2)
    kfloglik(0.0, s0, g0, g1, 1.0, sigmae, 0, 1, 0.3, data[:y])
end

                   proposal = Normal)

for i in 1:n
